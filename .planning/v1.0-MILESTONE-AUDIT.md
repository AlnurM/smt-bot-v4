# v1.0 Milestone Audit

## Executive Summary

All 6 phases are code-complete with 91 requirements implemented and ~150+ tests passing. However, the integration checker found a **critical cross-phase wiring gap**: the Signal ORM row is never created in the strategy scan pipeline, which means the Confirm → Order Execution → Position Monitoring flow is unreachable in live operation. This blocks ~16 downstream requirements.

Two additional gaps: the daily loss circuit breaker's `starting_balance` is never populated (financial safety defect), and two risk control functions (`check_rr_ratio`, `validate_liquidation_safety`) are implemented but never called.

## Critical Gaps (Must Fix)

### Gap 1: Signal DB Row Never Created (Severity 1)

**Location:** `bot/strategy/manager.py` ~line 275
**Impact:** SIG-01..06, TG-03, TG-04, ORD-01..05, MON-01..05 (~16 requirements)
**Root cause:** `generate_signal()` returns a dict, but no code creates a `Signal(...)` ORM row and commits it to get a real DB UUID before calling `send_signal_message()`. The dispatch uses a placeholder UUID `"00000000-..."`, so Confirm/Reject callbacks can never find the signal.

**Fix:** In `run_strategy_scan()`, after `generate_signal()` returns non-None:
1. Create `Signal(symbol=..., direction=..., entry_price=..., ...)` ORM instance
2. `session.add(signal_row)` + `await session.flush()` to get UUID
3. Set `signal["id"] = str(signal_row.id)` before calling `send_signal_message()`
4. After dispatch, store `signal_row.telegram_message_id` and `signal_row.zones_data`
5. `await session.commit()`

### Gap 2: DailyStats.starting_balance Never Set (Severity 2)

**Location:** `bot/monitor/position.py` ~line 194 (INSERT path of upsert)
**Impact:** RISK-04 (daily loss circuit breaker), TG-20 (80% warning)
**Root cause:** The `pg_insert(DailyStats)` INSERT clause never sets `starting_balance`. Falls back to `1.0` in executor, making breaker trigger on any ~$0.05 loss.

**Fix:**
1. In the INSERT path, fetch balance from Binance and set `starting_balance`
2. Call `check_and_warn_daily_loss()` from `_handle_position_close()` after DailyStats upsert

### Gap 3: Risk Controls Not Wired (Severity 3)

**Location:** `bot/risk/manager.py` — `check_rr_ratio()` and `validate_liquidation_safety()`
**Impact:** RISK-03, RISK-08

**Fix:**
1. Call `check_rr_ratio()` in `manager.py` before `send_signal_message()` — skip signal if R/R too low
2. Call `validate_liquidation_safety()` in `executor.py` before placing order — reject if liquidation too close
